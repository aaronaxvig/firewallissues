## GPC-21389

Fixed an issue where, when the GlobalProtect for Government app was installed on Android devices with FIPS-CC mode enabled, the application erroneously searched for SERVERAUTH EKU instead of CLIENTAUTH EKU, leading to the failure of client certificate verification.

## GPC-21707

Fixed an issue where, when the GlobalProtect app was installed on iOS platforms, users were not logged out when refreshing the connection or signing out. Additionally, the logs showed that the gateway-logout and gateway-config-release events did not occur as expected for the Prisma Access internal gateway.

## GPC-20071

Fixed an issue where, when the GlobalProtect app is installed on Android devices, the Host ID column of the Strata Logging Service (formerly CDL) log did not display the mobile-id setting value.
